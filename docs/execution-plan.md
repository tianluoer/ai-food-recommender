# 分阶段执行计划

> 最后更新：2026-07-22
> 版本：v1.0

---

## 总览

项目分 5 个 Phase 推进。每个 Phase 完成后进行验证，确认无误再进入下一 Phase。

---

## Phase 1：项目骨架搭建

**目标**：创建所有基础文件，确保项目结构正确，本地可启动

### 任务
1. 创建 `package.json`，添加 `openai` SDK 依赖
2. 创建 `vercel.json`，配置 `/api/chat` 路由
3. 创建 `index.html` 骨架（空白页面 + 基本 meta 标签）
4. 创建 `api/chat.js` 骨架（返回 mock 数据的函数）
5. 本地运行 `vercel dev` 验证项目可启动

### 验证标准
- ✅ `vercel dev` 正常启动
- ✅ 浏览器访问 `localhost:3000` 看到空白页面
- ✅ `POST /api/chat` 返回 mock 数据

### 产出文件
- `package.json`
- `vercel.json`
- `index.html`（骨架版）
- `api/chat.js`（骨架版）

---

## Phase 2：后端 API 实现

**目标**：实现与 DeepSeek API 的完整交互逻辑

### 任务
1. 编写 System Prompt（参考 requirements.md 中的问题策略）
2. 实现 `/api/chat` 逻辑：
   - 接收对话历史
   - 判断当前轮次
   - 调用 DeepSeek API
   - 解析并返回结果
3. 实现错误处理（超时、API 异常、无效请求）
4. 本地测试：用 curl 或 Postman 模拟 10 轮对话

### 验证标准
- ✅ 第 1 轮返回一个合理的问题
- ✅ 第 10 轮返回推荐结果（含菜系、菜品、理由）
- ✅ API 异常时返回错误信息而非崩溃

### 产出文件
- `api/chat.js`（完整版）

---

## Phase 3：前端 UI 实现

**目标**：完成完整的聊天界面，实现用户交互流程

### 任务
1. 实现 HTML 结构（顶栏、聊天区、进度条、输入区）
2. 实现 CSS 样式（参考 design-spec.md）
   - 聊天气泡样式
   - 响应式布局
   - 基础颜色和字体
3. 实现 JS 交互逻辑：
   - 发送消息 → 显示用户气泡 → 调用 API → 显示 AI 气泡
   - 进度条更新
   - 完成推荐后显示卡片
4. 实现基础状态处理：加载中、空状态、错误重试

### 验证标准
- ✅ 手机端和桌面端均正常显示
- ✅ 完成完整 10 轮对话流程
- ✅ 错误时显示提示并可重试

### 产出文件
- `index.html`（完整版）

---

## Phase 4：动画与体验打磨

**目标**：加入动画效果，提升整体体验

### 任务
1. 实现打字机效果（AI 文字逐字显示）
2. 实现气泡入场动画
3. 实现进度条填充动画
4. 实现加载动画（三个点跳动）
5. 实现推荐结果卡片动画
6. 实现食物 emoji 飘落欢迎效果
7. 实现「再来一次」重置功能
8. 细节优化：自动滚动、焦点管理、键盘发送

### 验证标准
- ✅ 动画流畅，无卡顿
- ✅ 所有交互动画符合 design-spec.md 规范
- ✅ 「再来一次」可完整重置流程

### 产出文件
- `index.html`（动画增强版）

---

## Phase 5：部署上线

**目标**：部署到 Vercel，分享链接可用

### 任务
1. 创建 GitHub 仓库
2. 推送代码到 GitHub
3. 在 Vercel 导入项目
4. 设置 `DEEPSEEK_API_KEY` 环境变量
5. 触发首次部署
6. 线上功能验证（完整 10 轮对话）
7. 获取公开链接并分享

### 验证标准
- ✅ 公开链接可正常访问
- ✅ 完整对话流程线上通过
- ✅ 手机端（微信内置浏览器）正常使用
